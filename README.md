# 法律構造化モノレポ

法律を超えた用語の共通性を活かすため、単一リポジトリで管理する。

## 構成

```
法律/        法文本体（01-kenpo, 02-doukou-hou, ...）
サブ/        補足・抄録（npo, ...）
メタ/        横断構造（graph, agent）
```

## メタ/

### graph/

- `terms.json` — 用語定義グラフ（24ノード・28エッジ）

- `committees.json` — 審議委員会マップ（4委員会 + 組織グラフ）
- `shocho.json` — 省庁グラフ（内閣府・文科省・経産省・国交省・警察庁の所管関係）
- `npo.json` — NPO法マッピング（NPO法人種別・所管・税制接続）

### agent/

- `world-model.json` — 世界モデル（人工衛星・射場・事業者等のエンティティと状態遷移）
- `compliance-rules.json` — コンプライアンスルール（6ルール: 平和利用・安全・許可・賠償・環境・国際）
- `agent-graph.json` — エージェント行動グラフ（入力→解析→照合→判定→出力）

## なぜモノレポか

単語の用法は法律間で共通のはずだから。

「道路」「自動車」「歩行者」「運転」「免許」などの用語は、
憲法・道路交通法・刑法・民法を横断して同じ概念を指す。
別々のリポジトリに散らばると、用語定義の一貫性を検証できない。

モノレポにすることで：

- 用語の横断検索が容易
- 定義の衝突・重複を発見できる
- 法体系全体を俯瞰できる
